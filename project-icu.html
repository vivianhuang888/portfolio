healthcare analytics case study

Bridging the Gap: Forecasting ICU Demand and Readmission Risk

A unified machine learning framework linking ED triage risk, ICU discharge safety, and short-term bed demand forecasting.

Duration
Fall 2025
Skills
Data Preprocessing (Python), Machine Learning (XGBoost), Data Visualization, Clinical Data Analytics (MIMIC-IV)

Overview / Description

Emergency department crowding and ICU strain are major hospital operational challenges. Patients who appear stable in the ED can deteriorate quickly, while patients discharged too early from ICU may face dangerous readmission risk.

This project analyzed the MIMIC-IV EHR database across the full patient journey, from ED triage to ICU admission, discharge, and potential readmission, to build a unified framework for real-time ICU bed demand forecasting.

Objective / Success Metrics

  1. Early deterioration detection (Phase 1): Predict ICU admission within 12 hours of ED arrival using early triage data.
  2. Safe discharge assessment (Phase 2): Predict ICU readmission within 72 hours using clinical data prior to ICU discharge.
  3. Unified capacity forecasting: Combine both predictions with length-of-stay estimates to support a dynamic ICU bed demand dashboard.

Approach

1) Data integration & cohort selection

  • Joined records across ed_stays, admissions, transfers, and icustays using subject_id and hadm_id.
  • Built two cohorts: an upstream cohort for ED-to-ICU prediction and a downstream cohort for ICU readmission prediction.

2) Feature engineering & unstructured data handling

  • Normalized home medication text and mapped more than 1,000 drug names into 22 pharmacologic categories.
  • Standardized free-text pain scores and transformed the top 25 chief complaints into binary indicators.

3) Predictive modeling

  • Used XGBoost with scale_pos_weight to address class imbalance.
  • Prioritized AUC and Recall to improve sensitivity to high-risk patients.

Results / Key Findings

Model 1: 12-Hour ICU Admission

AUC0.96

Recall0.93

The model captured the large majority of patients requiring urgent intensive care using early ED data.

Model 2: 72-Hour ICU Readmission

AUC0.72

Recall0.52

Readmission remained harder to predict, but the model still provided a useful screening signal for discharge risk.

Impact / Conclusion / Recommendations

  • Built a proof-of-concept ICU bed demand dashboard operating at 12-hour intervals.
  • Extended the work from single-patient prediction to system-level capacity forecasting.
  • Provided a practical framework for anticipating near-term bed shortages, prioritizing triage, and reducing unsafe premature discharge.

Future direction: Incorporate a dedicated length-of-stay model, clinical notes, and physiologic time-series data to improve real-time deployment and readmission precision.
